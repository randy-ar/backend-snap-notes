# 3.3.3 Analisis Teknologi Gemini API

Analisis teknologi Gemini API pada penelitian ini bertujuan untuk mengevaluasi kapabilitas layanan kecerdasan buatan dari Google dalam mengimplementasikan sistem pemrosesan struk belanja otomatis pada aplikasi Snap Notes. Teknologi ini berperan krusial sebagai komponen inti yang mengubah data teks mentah hasil Optical Character Recognition (OCR) menjadi format data JSON yang terstruktur dan kontekstual. Transformasi data ini menjadi fundamental karena hasil pemindaian struk belanja umumnya tidak memiliki struktur yang baku dan mengandung berbagai format penulisan yang bervariasi antar merchant. Untuk mengoptimalkan kemampuan analisis dari Large Language Model (LLM), sistem menggunakan teknik prompt engineering yang mengintegrasikan informasi koordinat spasial dari setiap teks. Pendekatan ini memberikan konteks posisional yang memungkinkan Gemini AI untuk memahami hubungan antar elemen teks, sehingga dapat mengklasifikasikan informasi seperti nama barang, kuantitas, harga satuan, dan subtotal dengan tingkat akurasi yang signifikan lebih tinggi.

## Penjelasan Alur Proses

### 1. Penerimaan Data OCR
Proses dimulai saat sistem menerima data teks hasil pemindaian OCR beserta informasi koordinat spasial dari setiap elemen teks pada struk belanja. Data ini mencakup posisi X, Y, lebar, dan tinggi dari setiap kata atau angka yang terdeteksi.

### 2. Server Snap Notes Membuat Prompt dengan Konteks Spasial
Server Snap Notes memproses data input dengan membuat prompt khusus yang mengintegrasikan informasi koordinat spasial ke dalam instruksi untuk Gemini AI. Prompt ini dirancang untuk membantu model AI memahami layout struk dan hubungan posisi antar elemen teks seperti nama barang, harga, dan jumlah.

### 3. Server Mengirim Request ke Gemini API
Server Snap Notes mengirimkan request HTTP ke Gemini API melalui Google Generative AI SDK. Request ini berisi prompt yang telah dikonfigurasi beserta data teks OCR dan informasi koordinat spasial untuk diproses oleh model AI.

### 4. Gemini API Menganalisis dengan Model AI
Gemini API menerima request dan melakukan analisis komprehensif menggunakan Large Language Model (gemini-2.5-flash). AI mengidentifikasi pola struktur struk, mengklasifikasikan informasi ke dalam kategori yang sesuai, dan memahami hubungan antar elemen berdasarkan posisi spasialnya.

### 5. Server Menerima Response JSON dari Gemini API
Server Snap Notes menerima response dari Gemini API dalam format JSON yang berisi data terstruktur tentang struk belanja. Response ini mencakup informasi merchant, daftar item belanja dengan harga dan jumlah, serta total pembayaran.

### 6. Server Memvalidasi Struktur Data JSON
Server Snap Notes melakukan validasi terhadap struktur dan kelengkapan data JSON yang diterima dari Gemini API. Validasi memastikan semua field wajib seperti nama toko, tanggal, total, dan array item telah terisi dengan format dan tipe data yang benar.

### 7. Selesai
Data struk yang telah berhasil diproses dan divalidasi siap digunakan dalam aplikasi Snap Notes untuk disimpan ke database dan ditampilkan kepada pengguna.

## Tabel 3.3 Spesifikasi *Request* pada Gemini API

Berikut adalah tabel spesifikasi *Request* yang dikirim oleh Server Snap Notes ke Gemini API dapat dilihat pada Tabel 3.3:

| **Spesifikasi *Request*** | |
|:--------------------------|:--|
| **URL** | `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent` |
| **Method** | POST |
| **Format** | application/json |
| **Header** | 1. Content-Type: application/json <br> 2. x-goog-api-key: {GEMINI_API_KEY} |
| **Parameter** | 1. model (path parameter) <br> 2. contents (body — berisi prompt teks OCR beserta koordinat spasial) |
| **Contoh Request** | Lihat blok kode di bawah |
