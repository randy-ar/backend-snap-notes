# 3.3.5 Analisis NestJS

Analisis pada subbab ini bertujuan untuk menggambarkan peran NestJS sebagai server backend yang berfungsi sebagai jembatan utama antara aplikasi mobile dengan berbagai layanan eksternal. NestJS bertanggung jawab mengelola autentikasi pengguna, menyimpan data struk belanja ke database dan storage, serta menjadi penghubung komunikasi dengan Gemini API untuk pemrosesan data OCR dan Supabase service untuk manajemen data pengguna.

## Penjelasan Alur Flowchart

Alur proses pada flowchart di atas terdiri dari beberapa tahapan proses utama yang dilakukan oleh NestJS sebagai jembatan antara aplikasi mobile dan layanan eksternal:

- **Aplikasi mengirim request HTTP ke NestJS Server**: Aplikasi mobile mengirimkan request yang berisi data struk belanja ke server NestJS untuk diproses.

- **NestJS menerima request dan validasi autentikasi via Supabase Auth**: Server NestJS menerima request dan melakukan validasi autentikasi pengguna melalui Supabase Auth untuk memastikan bahwa request berasal dari pengguna yang terautentikasi.

- **Controller routing ke module yang sesuai**: Setelah autentikasi berhasil, request diarahkan ke controller yang sesuai berdasarkan endpoint yang dipanggil untuk diproses lebih lanjut.

- **Pre-processing data OCR dari Google ML Kit**: NestJS melakukan pre-processing terhadap data hasil OCR dari Google ML Kit yang berisi teks mentah dan informasi posisi setiap baris teks dalam gambar struk.

- **Mengorganisasi informasi posisi teks**: Informasi posisi teks dari data OCR diorganisasi untuk membantu memahami struktur layout struk dan membedakan kolom item, jumlah, dan harga.

- **Bentuk prompt khusus untuk Gemini AI**: NestJS membentuk prompt khusus yang berisi instruksi jelas untuk Gemini AI agar dapat memahami struktur struk belanja, termasuk aturan untuk mengidentifikasi nama toko, tanggal, daftar item, harga, dan total pembayaran, serta informasi tambahan tentang posisi teks.

- **Service kirim ke Gemini API**: Prompt yang telah dibentuk dikirim ke Gemini API untuk diproses dan diubah menjadi struktur data yang terstruktur.

- **Gemini API mengembalikan data JSON terstruktur**: Gemini API memproses data OCR dan mengembalikan data dalam format JSON yang berisi informasi terstruktur seperti nama toko, tanggal belanja, daftar item, dan total harga.

- **Service upload gambar ke Supabase Storage**: NestJS mengunggah gambar struk ke Supabase Storage untuk penyimpanan permanen dan mengambil URL gambar yang telah diunggah.

- **Service simpan data ke PostgreSQL via Prisma ORM**: URL gambar dan data hasil parsing dari Gemini API disimpan ke database PostgreSQL melalui Supabase Database menggunakan Prisma ORM.

- **Service return hasil ke Controller**: Setelah data tersimpan, service mengembalikan hasil pemrosesan ke controller untuk dikirim kembali ke aplikasi.

- **Controller kirim response JSON ke aplikasi**: Controller mengirimkan response HTTP berisi data yang telah diproses dalam format JSON ke aplikasi mobile untuk ditampilkan kepada pengguna.

Arsitektur ini memisahkan tanggung jawab secara jelas antara komponen-komponen yang ada, di mana NestJS bertugas mengatur alur data, Gemini API menangani pemrosesan teks OCR, dan Supabase menangani penyimpanan data dan gambar. Pemisahan ini memudahkan pengembangan, pengujian, dan maintenance sistem di masa mendatang.
